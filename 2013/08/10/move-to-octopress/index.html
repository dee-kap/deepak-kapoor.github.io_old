Move to Octopress

Aug 10th, 2013 | Comments

Anyone who moves to Octopress must write a post talking about the experience. This seems like an unwritten rule which I dare not disobey. So here it is. These are my notes related to my move from Wordpress to Octopress. First of all, I still think that Wordpress is a fantastic platform and I have enjoyed running my blog on Wordpress since 2008. So why the move? Being a techie type myself I always crave for new and simple ways of doing things. Octopress at present satisfies the craving.

Other than simplicity of Octopress there are also these reasons for me to ditch Wordpress.

Security

Wordpress is as secure as it can be. This means that there still are and always will be potential security weak points. With the technical soup Wordpress uses (PHP, MySql and plugins), the attack surface is broad and too exposed for hackers to have a try. With Octopress, there is no database and no server side code execution. There is a satisfaction in knowing that the attack surface is as small as it can be. I also use Github pages for hosting and if Github gets hacked then I’m sure there will be many smart people working on the problem immediately.

Performance

This is a big one for me. Ever since I moved to Octopress, I have noticed great increased in speed. The page load which used to take an average of 8 seconds has come down to less than 3 seconds. This in my opinion is a big win.

Should you move to Octopress?

It totally depends on what you want out of your blogging, and your comfort level with tinkering. Wordpress offers convenience like no other platform for all including techies and non-techies. Octopress on the other hand is best suited for those who code and are comfortable using Git.
